# Frontend — Car Dealership Inventory System

Split from `context.md` (2026-07-10). Stack: FARMN frontend portion (exact mapping TBD — confirm with user before implementation).

## Requirements

- Modern single-page application.
- Registration and login forms.
- Dashboard/homepage displaying all available vehicles.
- Vehicle search and filtering (make, model, category, price range).
- Purchase button per vehicle; disabled when quantity is zero.
- Admin UI for adding, updating, and deleting vehicles (delete/restock are admin-only).
- Visually appealing, responsive, pleasant to use.
